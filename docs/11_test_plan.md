# 11 — 测试计划

## 测试层次

```
┌─────────────────────────────┐
│    手工验收测试（双机）        │  ← 发布前必须
├─────────────────────────────┤
│    集成测试（QtTest）         │  ← 发现→消息→传输→共享 串联
├─────────────────────────────┤
│    单元测试（QtTest）         │  ← storage / core / network
└─────────────────────────────┘
```

## 单元测试

使用 QtTest 框架。测试文件放在 `src/tests/` 下，CMake 通过 `add_test()` 注册。

| 测试模块 | 覆盖内容 | 测试文件 |
|---|---|---|
| storage | Database 初始化、迁移、CRUD 操作 | `tests/tst_storage.cpp` |
| models | JSON 序列化/反序列化 | `tests/tst_models.cpp` |
| Protocol | 消息打包/解包、边界情况（空消息、超长消息） | `tests/tst_protocol.cpp` |
| core | Beacon 发现逻辑（mock UDP）、Signal 消息路由 | `tests/tst_core.cpp` |

运行：`cmake --build build && ctest --test-dir build`

## 集成测试

| 场景 | 步骤 |
|---|---|
| 发现→聊天 | 两台设备启动 → 互见在线 → 发文本消息 → 双方确认收到 |
| 文件传输 | A 拖拽 100MB 文件到聊天 → B 接收 → 校验 SHA-256 → 文件完整 |
| 共享目录 | A 添加共享 → B 浏览目录 → 下载 50MB 文件 → 校验完整 |
| 授权 | B 首次访问 A 共享 → A 弹窗允许 → B 可浏览；A 拒绝 → B 看到拒绝 |
| 传输中断 | 传输中拔网线 → A 端任务状态变为"失败" → B 端文件不完整 |

## 异常测试

| 场景 | 预期行为 |
|---|---|
| 传输中发送方断网 | 发送方任务 Failed，接收方文件不完整 |
| 传输中接收方断网 | 同上 |
| 传输中接收方磁盘满 | 发送方收到 DISK_FULL 错误，任务 Failed |
| 发送 10GB 超大型文件 | 不崩溃，进度正确，1GB 分块稳定 |
| 同网段 10 台设备同时在线 | 发现列表正确，消息不受干扰 |
| Windows 防火墙开启 + 未放行 | 首次启动弹出防火墙提示，用户允许后正常工作 |
| UDP 广播被交换机隔离 | 手动添加 IP 可正常通信 |

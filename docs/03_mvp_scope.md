# 03 — MVP 功能清单与验收标准

> 本文档是 Agent 实施的核心参考。所有描述必须可测试、无歧义。
> 详细产品背景见 [02_prd.md](./02_prd.md)，UI 交互细节见 [08_ui_spec.md](./08_ui_spec.md)。

---

## P0 功能（首发必做）

### F001 — 跨平台构建与启动

| 项 | 内容 |
|---|---|
| 所属模块 | app/platform |
| 功能描述 | 在 Windows、Ubuntu/Debian、openKylin x86_64、macOS 上均能通过 CMake 构建并启动主窗口 |
| 验收标准 | 1. `cmake --build .` 在四个目标平台上均无错误完成。2. 启动后显示主窗口（允许空白壳子）。3. 窗口标题为"烽燧 FengSui"。4. 应用图标为占位图标。 |

### F002 — 首次启动向导

| 项 | 内容 |
|---|---|
| 所属模块 | ui/onboarding |
| 功能描述 | 用户首次运行时进入 3 步向导：设备命名 → 网络发现开关 → 默认下载目录 |
| 状态覆盖 | 正常完成、中途关闭窗口（下次启动重新进入向导）、已完成向导（不再显示） |
| 验收标准 | 1. 首次启动自动弹出向导窗口。2. 第 1 步：输入昵称（默认取系统主机名），点击"下一步"。3. 第 2 步：网络发现默认开启，可关闭，点击"下一步"。4. 第 3 步：选择默认下载目录（默认系统下载文件夹），点击"完成"。5. 完成后主窗口打开，向导不再出现。6. 设置值写入 SQLite。 |

### F003 — 局域网设备自动发现

| 项 | 内容 |
|---|---|
| 所属模块 | core/beacon |
| 功能描述 | 通过 UDP 广播在局域网内自动发现其他运行中的客户端，显示在线设备列表 |
| 状态覆盖 | 设备上线（新出现）、设备离线（心跳超时）、列表空态（无设备时显示提示）、发现关闭（手动关闭后不广播也不接收） |
| 验收标准 | 1. 两台同网段 Windows 电脑启动后，5 秒内互见对方。2. 一台 Windows 与一台 Linux 同网段启动后，5 秒内互见。3. 关闭一台客户端，另一台 15 秒内将其标记为离线。4. 联系人列表显示在线设备昵称、设备名和在线状态。5. 断开网络后，已发现设备在超时后标记离线。6. 恢复网络后，设备重新上线并恢复在线状态。 |

### F004 — 手动添加 IP 连接

| 项 | 内容 |
|---|---|
| 所属模块 | core/beacon |
| 功能描述 | 当 UDP 发现不可用时，允许用户手动输入 IP:端口 添加设备 |
| 验收标准 | 1. 输入合法 IP 和端口后点击添加，尝试 TCP 连接。2. 连接成功则设备出现在列表中。3. IP 格式错误时给出提示。4. 连接失败时显示"无法连接"。 |

### F005 — 单聊文本消息

| 项 | 内容 |
|---|---|
| 所属模块 | core/signal, ui/chat |
| 功能描述 | 点击在线设备进入聊天窗口，发送和接收文本消息，消息持久化到 SQLite |
| 状态覆盖 | 发送中（网络中断时等待）、发送成功、发送失败（对方离线或网络不通）、对方正在输入提示（P1）、消息时间戳 |
| 验收标准 | 1. 点击在线设备 → 打开聊天窗口。2. 输入文本 → 回车或点击发送 → 消息出现在双方窗口。3. A 发送的消息在 B 窗口实时出现，延迟 < 1 秒。4. 聊天记录写入 SQLite，关闭重开后历史可见。5. 对方离线时发送，显示"发送失败"。6. 聊天窗口显示对方在线/离线状态。 |

### F006 — 文件拖拽发送

| 项 | 内容 |
|---|---|
| 所属模块 | core/courier, ui/chat |
| 功能描述 | 拖拽单个文件到聊天窗口即可发送，支持大文件（≥1GB），接收方可选择接受或拒绝 |
| 状态覆盖 | 等待对方接收、对方拒绝、传输中（含进度百分比和速度）、传输完成、传输失败（含网络中断和对方取消） |
| 验收标准 | 1. 拖拽一个文件到聊天窗口，自动发起传输请求。2. 接收方弹出"接收/拒绝"提示。3. 拒绝后发送方显示"对方已拒绝"。4. 接收后显示实时进度（百分比 + 速度）。5. 传输完成后文件保存到接收方默认下载目录。6. 传输记录写入 SQLite（文件名、大小、发送方、时间、状态）。7. 1GB 文件可在千兆局域网内完成传输，不崩溃。8. 传输中途断网，发送方任务状态变为"失败"。 |

### F007 — 文件夹传输

| 项 | 内容 |
|---|---|
| 所属模块 | core/courier, ui/chat |
| 功能描述 | 拖拽文件夹到聊天窗口，自动递归打包传输 |
| 验收标准 | 1. 拖拽文件夹 → 显示包含的文件数量和总大小 → 确认后传输。2. 接收方收到后按原目录结构保存。 |

### F008 — 传输中心

| 项 | 内容 |
|---|---|
| 所属模块 | ui/transfer_center, core/courier |
| 功能描述 | 统一面板查看所有进行中、已完成、失败的文件传输任务 |
| 状态覆盖 | 进行中、已完成、已取消、失败（含错误原因） |
| 验收标准 | 1. 传输中心显示全部传输记录（发件/收件均可）。2. 进行中的任务显示进度。3. 已完成的任务点击可打开文件所在目录。4. 失败的任务显示失败原因。5. 列表可按时间/状态排序。 |

### F009 — 共享文件夹

| 项 | 内容 |
|---|---|
| 所属模块 | core/share, ui/share |
| 功能描述 | 用户指定本地目录为共享，同网段其他人可浏览目录结构和下载文件 |
| 验收标准 | 1. 在我的共享页点击"添加共享" → 选择本地目录 → 共享生效。2. 共享页列出当前共享目录，可移除。3. 其他用户浏览共享文件页时，看到所有在线的共享源。4. 点击共享源 → 浏览其共享目录的文件列表（文件名、大小、修改时间）。5. 点击文件 → 下载到本地默认下载目录。6. 支持子目录浏览。 |

### F010 — 共享访问授权

| 项 | 内容 |
|---|---|
| 所属模块 | core/share, ui |
| 功能描述 | 其他用户首次访问我的共享目录时，弹出授权确认窗口 |
| 验收标准 | 1. 其他人首次浏览我的共享目录 → 我这端弹窗"XXX 请求访问您的共享文件"。2. 点击"允许"→ 对方可浏览下载。3. 点击"拒绝"→ 对方看到"访问被拒绝"。4. 允许后，该设备在本次会话内不再弹窗。5. 允许记录写入 SQLite，下次同设备访问可选"记住选择"。 |

### F011 — 网络诊断

| 项 | 内容 |
|---|---|
| 所属模块 | ui/settings, core/diagnostics |
| 功能描述 | 自检页面，帮助用户排查发现和连接问题 |
| 验收标准 | 1. 显示本机所有 IP 地址和网卡信息。2. 显示当前监听端口。3. 显示防火墙状态提示（Windows 检测 Windows Defender 防火墙规则）。4. 显示"UDP 广播测试"结果（发送测试包并检查是否收到回包）。5. 显示当前在线设备数。 |

## P1 功能（后续增强）

### F012 — 广播通知

| 项 | 内容 |
|---|---|
| 所属模块 | core/signal, ui/broadcast |
| 功能描述 | 向全网段发送广播通知，可选择要求确认收到 |
| 验收标准 | 1. 发送广播 → 所有在线设备收到通知弹窗。2. 可选择"需要确认"，接收方需点击"已收到"。3. 发送方可查看已读/未读列表。 |

### F013 — 文件投递箱

| 项 | 内容 |
|---|---|
| 所属模块 | core/dropbox, ui/dropbox |
| 功能描述 | 创建收件任务（命名 + 描述），协作者提交文件，自动统计提交情况 |
| 验收标准 | 1. 新建投递任务：填写任务名、描述、截止时间。2. 其他人看到投递任务列表 → 点击进入 → 提交文件。3. 创建者查看每个任务的提交列表和提交者。4. 支持勾选"已收到"标记。 |

### F014 — 断点续传

| 项 | 内容 |
|---|---|
| 所属模块 | core/courier |
| 功能描述 | 大文件传输中断后可从中断位置继续 |
| 验收标准 | 1. 传输中断后重试 → 从中断的字节偏移继续。2. 不需要重新传输已完成的部分。 |

### F015 — 浏览器访客模式

| 项 | 内容 |
|---|---|
| 所属模块 | core/share |
| 功能描述 | 未安装客户端的用户可通过浏览器访问共享文件和投递 |
| 验收标准 | 1. 本机启动内建 HTTP 服务。2. 同网段浏览器输入 IP:端口 → 查看共享目录和文件。3. 可下载文件。4. 可提交文件到投递箱。 |

### F016 — 系统托盘增强

| 项 | 内容 |
|---|---|
| 所属模块 | ui/tray |
| 功能描述 | 关闭主窗口后最小化到托盘，托盘图标显示在线状态，右键菜单操作 |
| 验收标准 | 1. 关闭主窗口 → 最小化到系统托盘而非退出。2. 托盘图标随在线状态变化（在线绿色、离线灰色）。3. 右键菜单：在线/隐身切换、打开主窗口、退出。4. 有新消息托盘图标闪烁或有气泡提示。 |
